NASFAA Data Sharing Decision Tree — methodology, engineering, and sources

1. What this is

One-paragraph description of the two interactive pages (walkthrough + quiz) and what question they answer.

2. Why it exists

Motivation: the NASFAA PDF is authoritative but linear; an interactive tree + scenario quiz makes the rules navigable for aid administrators, registrars, and student-success staff.

3. How it works

  • Walkthrough — yes/no traversal of the decision DAG, one question at a time, ending at a permit/deny terminal with citation.
  • Quiz — 24 real-world scenarios; user predicts permit/deny; reveal shows the firing rule, citation, contextual flavor, and case studies.

4. Methodology

  • 4.1 Rule extraction from the canonical PDF — how each box in the NASFAA decision tree became an entry in nasfaa_rules.yml.
  • 4.2 YAML as single source of truth — rules + question DAG + scenarios all live in YAML; the engine, CLI, and web pages read from there.
  • 4.3 Dual-engine cross-verification — a declarative RuleEngine and an imperative DecisionTree implement the same logic two different ways; an exhaustive spec verifies they agree on every input combination.
  • 4.4 Exhaustive scenario library — 24 named scenarios cover every rule in the tree; each is checked against both engines on every CI run.

5. Engineering notes

  • 5.1 Frontend tooling choices — why Make over Rake/just/npm; why node:test + --experimental-test-coverage over Vitest; why no TypeScript port.
  • 5.2 Coverage survey tool — bin/coverage-survey reads SimpleCov resultsets and walks the JS tree, flagging weak coverage + duplicate utilities in one report.
  • 5.3 Quiz case-study research — parallel research subagents (one per regulatory area) drafted case-study summaries with strict WebFetch URL verification; main thread did the editorial pass + rewrites.
  • 5.4 Test coverage philosophy — what "100%" buys (regression protection on every code path), what it doesn't (correctness against the PDF), and why both engines exist.

6. Sources & primary authorities

  • NASFAA Data Sharing Decision Tree (PDF) — the canonical source the rules were extracted from.
  • IRC §6103(l)(13) — Federal Tax Information disclosure for student financial aid.
  • FERPA — 20 USC §1232g; 34 CFR Part 99.
  • HEA §483 — FAFSA / FUTURE Act consent and use restrictions.
  • DCL GEN-25-08 and related Dear Colleague Letters cited in the quiz reveals.

7. Limitations & disclaimer

Explicit statement that this is not legal advice, that the rule set reflects a point-in-time reading, and that institutions should verify against current regulations before acting.

Who built it, the GitHub repository, how to file issues against specific rules or scenarios.
